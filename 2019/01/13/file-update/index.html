apache解析漏洞（1.x、2.x）会从右往左依次解析后缀名 如404.asp.jpg会->404.asp 但是php可能还会再次校验后缀造成不解析
`目录为.asp结尾的会把里面的文件都解析成asp文件

->解释：apache运行php时有两种运行模式：
fast-cgi–>以接口方式，后续还会校验后缀
module–>php作为apache一部分，后缀名不再解析
